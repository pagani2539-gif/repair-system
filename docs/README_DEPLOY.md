# คู่มือการติดตั้งและ Deploy ระบบซ่อมบำรุงและเคลมสินค้า (Repair & Claim System)

คู่มือฉบับนี้อธิบายขั้นตอนการติดตั้ง ตั้งค่า และเปิดใช้งานระบบสำหรับใช้จริง (Production) หรือใช้สำหรับการพัฒนา (Development) โดยละเอียด

---

## 📋 ภาพรวมสถาปัตยกรรมระบบ (System Architecture)
ระบบนี้ได้รับการออกแบบด้วยโครงสร้างแบบ **SPA (Single Page Application)** ร่วมกับ **Restful API**:
- **Frontend (Client):** เขียนด้วย React 19 + TypeScript + Vite รันที่พอร์ตพัฒนา `5173` (ในโหมด Development)
- **Backend (Server):** เขียนด้วย Node.js + Express + SQLite (รันที่พอร์ต `5221`)
- **การเชื่อมต่อ Asset & API:** ปรับปรุงใหม่ให้ทำงานแบบ Dynamic ยืดหยุ่นตาม IP/Domain ที่เปิดใช้งานจริงโดยอัตโนมัติ

---

## 🛠️ 1. เตรียมความพร้อมก่อนติดตั้ง (Prerequisites)
ก่อนเริ่มดำเนินการติดตั้ง โปรดตรวจสอบว่าเครื่องเซิร์ฟเวอร์หรือคอมพิวเตอร์ที่จะรันระบบได้ติดตั้งโปรแกรมเหล่านี้แล้ว:
1. **Node.js (LTS version):** เวอร์ชัน 18 ขึ้นไป
2. **npm:** มาพร้อมกับตัวติดตั้ง Node.js
3. **PM2 (สำหรับใช้งานจริง):** ติดตั้งในระดับ Global เพื่อจัดการระบบแบบ Background
   ```bash
   npm install -g pm2
   ```

---

## ⚙️ 2. ขั้นตอนการเตรียมโฟลเดอร์และการติดตั้ง Dependencies
คัดลอกโฟลเดอร์โปรเจกต์ทั้งหมดไปยังเครื่องเซิร์ฟเวอร์ จากนั้นเปิด Terminal หรือ Command Prompt เข้าไปยังโฟลเดอร์หลักของโปรเจกต์:

### ขั้นตอนที่ 2.1: ติดตั้งไลบรารีฝั่งเซิร์ฟเวอร์ (Backend)
```bash
cd server
npm install
```

### ขั้นตอนที่ 2.2: ติดตั้งไลบรารีฝั่งหน้าเว็บ (Frontend)
```bash
cd ../client
npm install
```

---

## 🗄️ 3. การเตรียมระบบฐานข้อมูล (Database Setup)
ระบบนี้ทำงานร่วมกับฐานข้อมูล **SQLite3** ซึ่งมีความเสถียรและไม่ต้องรัน Service ภายนอกเพิ่มเติม

### กรณีที่ 1: สำหรับการติดตั้งระบบใหม่แกะกล่อง
จากโฟลเดอร์เซิร์ฟเวอร์ (`server`) ให้รันคำสั่งเพื่อสร้างตารางทั้งหมดที่จำเป็นขึ้นมาใหม่โดยอัตโนมัติ:
```bash
npm run init-db
```
*ระบบจะสร้างไฟล์ฐานข้อมูลไว้ที่ `server/database/repair_system.db`*

### กรณีที่ 2: สำหรับเครื่องที่มีไฟล์ฐานข้อมูลเดิมและต้องการอัปเกรด (Migrations)
หากมีฐานข้อมูลเดิมอยู่แล้วและต้องการเพิ่มฟิลด์ใหม่ (เช่น `is_read`, `type`, `received_at`) ให้รันไฟล์ Script ที่เตรียมไว้จากโฟลเดอร์หลัก:
```bash
cd ..
node scripts/migrate.js
```

---

## 📦 4. การ Build ระบบสำหรับใช้งานจริง (Production Build)
ก่อนเปิดให้ผู้ใช้งานทั่วไปเข้ามาใช้ ต้องทำการรวมไฟล์หน้าบ้านให้อยู่ในโหมด Production เพื่อประสิทธิภาพสูงสุด:

1. เข้าไปที่โฟลเดอร์หน้าบ้าน:
   ```bash
   cd client
   ```
2. รันคำสั่ง Build:
   ```bash
   npm run build
   ```
   *คำสั่งนี้จะทำการ Compile TypeScript และสร้างโฟลเดอร์ `client/dist` ขึ้นมา*

> [!NOTE]
> **ระบบหลังบ้านได้รับการตั้งค่าให้ตรวจจับไฟล์ Build หน้าบ้านอัตโนมัติ:**
> เมื่อเซิร์ฟเวอร์เริ่มทำงาน หากพบโฟลเดอร์ `client/dist` เซิร์ฟเวอร์ Express จะทำการเปิดหน้าเว็บและรับส่ง API ให้อยู่บนพอร์ตเดียวกัน (`5221`) ทันที ทำให้ประหยัดพอร์ต และผู้ใช้ไม่จำเป็นต้องตั้งค่า URL แยกใด ๆ

---

## 🚀 5. วิธีการเปิดรันระบบ (Running the System)

### 💻 การรันเพื่อพัฒนาต่อ (Development Mode)
หากต้องการรันเพื่อแก้ไขและเขียนโค้ดต่อ ให้เปิด Terminal แยกเป็น 2 หน้าต่าง:
- **หน้าต่างที่ 1 (Backend):** 
  ```bash
  cd server
  npm run dev
  ```
- **หน้าต่างที่ 2 (Frontend):** 
  ```bash
  cd client
  npm run dev
  ```
  *เปิดเบราว์เซอร์ไปที่ `http://localhost:5173` เพื่อเข้าใช้งาน*

---

### 🌐 การรันใช้งานจริงบนเซิร์ฟเวอร์ (Production Mode) ด้วย PM2
เมื่อทำการ Build ขั้นตอนที่ 4 เรียบร้อยแล้ว เราจะใช้ **PM2** เพื่อให้เซิร์ฟเวอร์รันอยู่เบื้องหลังตลอดเวลา (แม้ว่าจะปิดหน้าต่าง Terminal ไปแล้ว)

1. กลับไปที่โฟลเดอร์หลักของโปรเจกต์
2. เริ่มการทำงานด้วย PM2:
   ```bash
   pm2 start server/index.js --name "repair-system"
   ```
3. บันทึกสถานะโปรเซสเพื่อให้รันอัตโนมัติเมื่อเซิร์ฟเวอร์รีบูต:
   ```bash
   pm2 save
   pm2 startup
   ```

---

## 🖥️ 6. การเข้าใช้งานระบบและทดสอบ
เมื่อระบบรันเรียบร้อยแล้ว ผู้ใช้สามารถเปิดเบราว์เซอร์และเข้าหน้าเว็บผ่าน IP ของเซิร์ฟเวอร์ได้โดยตรง:
- **URL ตัวอย่าง:** `http://localhost:5221` หรือ `http://[IP-Address-ของเซิร์ฟเวอร์]:5221`
- **ระบบจะทำสิ่งต่อไปนี้ให้อัตโนมัติ:**
  - โหลดหน้าเว็บ React ขึ้นมาแสดงผลโดยอัตโนมัติ
  - เรียกใช้ API ไปที่หลังบ้านผ่าน Origin ปัจจุบัน ทำให้ไม่มีปัญหาการบล็อกจาก CORS
  - แสดงและดึงรูปภาพของใบแจ้งซ่อม/แจ้งเคลมได้ถูกต้อง แม้ว่าจะเข้าใช้งานจากเครื่องอื่นในวงแลน

## 🔍 7. การตรวจสอบการทำงานและแก้ไขปัญหา (Troubleshooting)

### คำสั่ง PM2 ที่จำเป็น:
- **ตรวจสอบสถานะระบบ:** `pm2 status`
- **ตรวจสอบ Logs แบบ Real-time:** `pm2 logs repair-system`
- **สั่ง Restart ระบบใหม่:** `pm2 restart repair-system`
- **สั่งหยุดรันระบบ:** `pm2 stop repair-system`

### ปัญหาที่พบบ่อย (Common Issues):
1. **พอร์ตชนกัน (Port 5221 in use):**
   หากขึ้น Error ว่ามีโปรเซสอื่นใช้งานพอร์ต `5221` อยู่ สามารถเปลี่ยนพอร์ตไปใช้พอร์ตอื่นได้โดยตั้งค่าตัวแปรระบบ (Environment Variable) เช่น:
   - บน Windows (CMD): `set PORT=8080 && pm2 start server/index.js --name "repair-system"`
   - บน Linux/macOS: `PORT=8080 pm2 start server/index.js --name "repair-system"`
2. **ภาพอัปโหลดไม่แสดง:**
   ตรวจสอบว่าโฟลเดอร์ `server/uploads` ได้รับสิทธิ์ในการอ่าน/เขียนไฟล์ (Read/Write Permission) และตรวจเช็คว่าพอร์ตใน Logs ไม่มีปัญหาใด ๆ
